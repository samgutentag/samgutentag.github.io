For this example we will be setting up an environment for a git project called 'gutentag'


Step 0. Create a nice clean directory for the project to live in
    $ mkdir ~/Developer/gutentag
    $ cd ~/Developer/gutentag


Step 1. Make sure conda is up to date
    $ conda update conda


Step 2. Create a conda env, with a few specifications
    $ conda create -n gutentag python=3.6


Step 3. activate the new env
    $ source activate gutentag


Step 4. Install some conda packages
    $ conda install -n gutentag [packages]


Step 5. Install any needed pip packages
    $ pip install [packages]


Step 6. Create special environment variables that load and unload with the environment such as an API Key
    Will need to create two additional files
    Navigate to conda envs directory, and make additional directories as follows

    $ cd ~/anaconda/envs/gutentag
    $ mkdir -p ./etc/conda/activate.d
    $ mkdir -p ./etc/conda/deactivate.d
    $ touch ./etc/conda/activate.d/env_vars.sh
    $ touch ./etc/conda/deactivate.d/env_vars.sh

    Edit the activate and deactivate files as such, adjust strings as such, example uses Twitter API Keys as a demo
    NOTE THE LACK OF SPACES IN ACTIVATE

    activate.d/env_vars.sh
        > #!/bin/sh
        >
        > export TWITTER_ACCESS_TOKEN='0123456789-abcdefg'
        > export TWITTER_ACCESS_TOKEN_SECRET='0123456789-abcdefg'
        > export TWITTER_CONSUMER_KEY='0123456789-abcdefg'
        > export TWITTER_CONSUMER_SECRET='0123456789-abcdefg'

    deactivate.d/env_vars.sh
        > #!/bin/sh
        >
        > unset TWITTER_ACCESS_TOKEN
        > unset TWITTER_ACCESS_TOKEN_SECRET
        > unset TWITTER_CONSUMER_KEY
        > unset TWITTER_CONSUMER_SECRET


Step 7. Navigate back to project directory
    $ cd ~/Developer/gutentag


Step 8. write an environment.yml file so we can rework from here
    $ conda env export | grep -v '^prefix: ' > ./environment.yml


Step 9. git-ify everything
    $ git init
    $ git add environment.yml && git commit -m 'initial commit'


Step 10. Deactivate and Reactivate environment. and make sure custom variables were loaded
    $ source deactivate
    $ source activate gutentag
    $ printenv
        Make Sure your variables are there!  typically towards the bottom


Step 11. (Optional) Terminal Aliases -> cds to the working directory, and activate the conda env
    Add the following line to your .aliases or .bash_profile file.

alias gutentag="cd ~/Developer/gutentag ; source activate gutentag"

If you are setting up on a second machine, lets assume you have already cloned the project from github


Step 0. Navigate to project directory
    $ cd ~/Developer/gutentag


Step 1. Make sure conda is up to date
    $ conda update conda


Step 2. Create env from environment.yml file
    $ conda env create -f environment.yml


Step 3. Verify the environment was created
    $ source activate gutentag
    $ conda list
